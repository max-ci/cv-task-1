You can add, remove and change content of the boxes on the right to see how component will behave when a user change something.

The features boxes on the left are pushed a little bit to the top and the right ones to the bottom to keep ideal vertical alignment with the left side of the container.

HTML markup is a little bit simpler using CSS Grid (here and there I don't have to use additional containers thanks to gap property, which exists also in CSS Flexbox but it's not super widely supported yet).

I could use the same HTML markup for CSS Flexbox without gap property, but CSS would be more complicated (e.g. width based on calc function and right margin every second box), not flexible enough for a component used by different people.
